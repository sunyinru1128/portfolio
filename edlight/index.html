Project OVERVIEW

Edlight

Edlight empowers teaching behavior based on students' performance in assignments.
Its goal: Give students feedback on their work, then assign intervention based on feedback; so data becomes teaching outcomes instead of judgment.

Before I joined the Edlight Team as a founding designer, Edlight only have MVP product. The old dashboard merely displayed the pure numbers of grades which could not tell teachers any insights about students\u2019 competency profiles.

Therefore, to transform students\u2019 grades into teaching guidance, my team redefined the new metric system (Competency standard) and I designed the data visualization dashboard based on the new metric.

User Testing & Iteration

# Iteration case 1

Create innovative layout to orchestra busy information

Before: Pure Table. Due to the complexity of competencies, table creates extreme reading loads.
After: Dual Tree. Combine "standard" tree with "student" table to provide flexible comparison options.

# Iteration case 2

Clarify the grouping logics based on teaching habits

Before: Complex filter structure. We thought grouping required very flexible options, turning it into a database-like search.
After: Tab view combined with filter. A tab view organizes major categories, mapping exactly to teachers' mental models.
